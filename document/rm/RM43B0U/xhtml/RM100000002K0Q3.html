VEHICLE CONTROL HISTORY (RoB)

DESCRIPTION

  • Vehicle control history (RoB) stores the ECU data (record data) when triggered by a specific vehicle behavior.
  • Checking history information and freeze frame data can be of use when performing troubleshooting in response to customer feedback.
  • Items such as the number of data that can be recorded, whether data is stored chronologically, the number of chronological data, freeze frame data items and the storage area for records in the ECU are all set for each storage group.
  • Freeze frame data items for vehicle control history (RoB) data are filtered down to the number of items required for each group, and items that have not changed within the chronological data are displayed as single points.

    The content of each item in freeze frame data will be mostly the same as the content in the normal Data List.

PRECAUTIONS FOR USE

  • Vehicle control history (RoB) is updated whenever a trigger occurs, therefore make sure to save data prior to checking symptoms.
  • Vehicle control history (RoB) might also be stored when the Active Test is performed or learning values, etc. are relearned during a procedure. In this case, clear the vehicle control history (RoB) before returning the vehicle.

CLEAR VEHICLE CONTROL HISTORY (RoB)

(a) According to the display on the GTS, check the vehicle control history (RoB).

Chassis > Adaptive Variable Suspension System > Utility

Tester Display

Vehicle Control History (RoB)

(b) Select the clear button and follow the instructions on the screen to clear the vehicle control history (RoB).

NOTICE:

When the vehicle control history (RoB) is cleared, all the stored history items are cleared.
